# 医学内容审校工具 — 使用说明

## 快速开始

1. **双击** `index.html` 文件，用浏览器打开（推荐 Chrome / Edge）
2. 把 `.txt` / `.md` / `.docx` / `.pptx` 文件**拖进上传区**，或点击选择
3. 自动解析并显示：
   - **概览统计**：问题总数、按等级/类别统计
   - **原文预览**：左侧，错误处高亮标色（🔴红/🟡黄/🔵蓝）
   - **问题清单**：右侧，分级分类列表
   - **双向联动**：点击清单项 ↔ 原文高亮块互相滚动定位

## 规则维护

直接编辑 `js/rules.js`，每条规则是一个对象，格式见文件内注释。

**关键字段**：
- `keywords` — 显式词表（优先，误报低）
- `pattern` — 正则表达式
- `map` — 替换映射表（key=错误写法，value=建议规范）
- `ignoreContext` — 否定白名单（命中这些上下文则跳过）
- `requireContext` — 正向条件（必须邻近这些词才报）
- `enabled` — `true`/`false` 开关

**示例**：
```js
{
  id: 'C1-001', category: 1, type: 'keyword', level: 'red',
  keywords: ['最佳', '最好', '最强'],
  ignoreContext: ['最近', '最终'],
  message: '绝对化用语「{match}」，违反《广告法》',
  suggestion: '删除或改为相对表述'
}
```

修改后**刷新浏览器**（Ctrl+Shift+R 强刷），重新上传文件即生效。

## 技术说明

- **纯前端**：所有处理在浏览器内存完成，文件不上传服务器
- **file:// 协议安全**：所有脚本用 `<script src>` 顺序加载，不依赖 ES module / fetch
- **UMD 双用**：`rules.js` 和 `engine.js` 同时支持浏览器全局变量和 Node require，方便测试
- **专业价值 = 词表质量**：规则库文档已明确，词表需医学编辑逐条审定（尤其药名/术语对照表）
